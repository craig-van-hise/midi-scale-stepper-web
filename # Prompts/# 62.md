# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**BUG FIX (DOUBLE INVERSION):** The visual Input Keyboard (`KeySplitKeyboard.tsx`) is incorrectly applying the inversion state to its click targets. Because the central engine (`useMidiStore.ts -> processStepperAction`) already handles the mathematical inversion, the UI applying it beforehand causes a double-negative (e.g., clicking +1 while inverted sends -1 to the store, which the store inverts *back* to +1).

**Objective:** Strip all inversion-awareness from the `KeySplitKeyboard.tsx` event handlers. The visual input keyboard must act as a "dumb" physical controller, passing the raw, unmodified physical key index to the store exactly like the Web MIDI hardware router does.

## 2. Technical Decisions & Dependencies
* **Single Source of Truth:** `useMidiStore.ts` is the sole authority on inversion math.
* **Dumb UI Trigger:** In `KeySplitKeyboard.tsx`, if a user clicks a key in the Stepper zone, the index passed to `processStepperAction` must strictly be `midiNote - stepperOffset` (where `stepperOffset` is `48` for 88-key mode or `60` for 49-key mode). It must NEVER read the store's inversion state to modify this index.
