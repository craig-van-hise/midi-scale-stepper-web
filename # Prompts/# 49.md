# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The Two-Octave Bidirectional Stepper (24-key layout) has been successfully implemented, but requires critical interaction refinements. 

**Current Issues to Resolve:**
1. **Silent Home Key:** The `HOME` action resets the pitch pointer in the store but fails to trigger an output note.
2. **Radio Button UI Bleed:** The utility keys (Invert, Home, Repeat) are currently inheriting the `selectedIndex` "radio button" highlight behavior intended only for Step/Octave keys. Utility keys must highlight independently: Toggles should stay on when active, and Momentaries (including Home/Repeat) should only light up while physically pressed.
3. **Typography:** Utility keys should use standard `lucide-react` icons instead of text labels to clean up the physical UI.

## 2. Technical Decisions & Dependencies
* **The "Silent Home" Fix:** The `HOME` action currently calls `triggerHomeReset()`, which updates `lastPlayedMidi` but does not calculate or route an output note. The fix is strictly sequential: When `HOME` is triggered, the store must call `triggerHomeReset()`, and *immediately* execute a `0` step offset via `executeEngineFn(0)` to force the engine to play and route the new home note.
* **Selection State Decoupling:** The UI component must stop blindly calling `setSelectedIndex(i)` on every key press. `selectedIndex` must *only* track the last played `STEP` or `OCTAVE`. 
* **Icon Mapping:** We will utilize `lucide-react` (already in the project) for the following action types:
  * `HOME` -> `<Home size={16} />`
  * `REPEAT_LAST` -> `<Repeat size={16} />`
  * `INVERT_TOGGLE` and `INVERT_MOMENTARY` -> `<ArrowUpDown size={16} />`
