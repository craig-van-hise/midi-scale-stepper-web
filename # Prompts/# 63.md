# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**TYPOGRAPHY STANDARDIZATION:** We are unifying the font architecture across the application to establish a clear visual hierarchy and eliminate inconsistent browser fallbacks. We are implementing a "2 + 1" design system: **Outfit** (Display), **Inter** (Sans/UI Data), and **Roboto Mono** (Tabular Data).

**Objective:** Update the inline `fontFamily` styles within `ScaleInspectorNotation.tsx` to explicitly map to the CSS variables defined in `index.css` (`--font-display`, `--font-sans`) and standard `Roboto Mono`, while strictly preserving the `Bravura` SMuFL assignments for musical glyphs.

## 2. Technical Decisions & Dependencies
* **Variable Mapping:** `index.css` exposes Tailwind v4 theme variables. 
  * `--font-display` resolves to Outfit.
  * `--font-sans` resolves to Inter.
* **Inline Style Updates:** Because `ScaleInspectorNotation.tsx` relies heavily on exact inline styles for absolute positioning and canvas rendering, we will inject these variables directly into the React `style` objects using the `var(--variable-name)` syntax.
* **Preservation:** Elements with `className="smufl"` or `fontFamily: 'Bravura, serif'` MUST NOT be altered.
