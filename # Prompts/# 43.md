# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The user has identified a "swallowed repetition" bug during legato playing. When alternating between a Stepper key and the Repeat Note (Step 0) while overlapping the keys (legato), the repeat note fails to play. 

**Root Cause Analysis:**
1. **React State Batching:** When the engine processes a unison repeat, it calculates the same target note, calls `removeOutputKey(62)`, and immediately calls `addOutputKey(62)` synchronously. React batches these state changes, so the `useEffect` hardware dispatcher in `useWebMidi.ts` never sees a delta in the `outputActiveKeys` array, resulting in no Note Off and no new Note On.
2. **Premature Release Cutoffs (Monophonic Overlap):** When the user releases the first overlapping key, the Note Off handler blindly deletes the target note from the output array, instantly cutting off the second key's note.

**Objective:** Implement an explicit 5ms staggered re-trigger for unison repetitions to bypass React's batching, and enforce a "Reference/Ownership Check" during Note Off events so a target note is only choked if no other physical keys are actively holding it.

## 2. Technical Decisions & Dependencies
* **Staggered Retriggering:** Inside `ScaleStepperEngine.ts`, if the target note is already playing, we will forcefully choke it, then wrap the `addOutputKey` in a `setTimeout(..., 5)` to spread the state change across two React render ticks, guaranteeing the hardware dispatcher fires both the Note Off and Note On.
* **Registry Ownership:** In `useWebMidi.ts`, before choking a note, we will iterate over `activeNotesRegistry.current` to ensure no *other* physical key maps to the same target note.

        ```typescript
          store.removeOutputKey(lastPlayedMidi);
        }

            }, 5);
          } else {
            // Standard execution
            store.addOutputKey(finalMidi);
          }
        }
        ```
