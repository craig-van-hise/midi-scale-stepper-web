

**Context & Goal:**
We have identified a state-bleed bug in `src/store/useMidiStore.ts`. The `syncKeySwitches` function is aggressively overwriting user-defined scale selections passively every time the active state changes. We need to remove this passive synchronization from the core setters, but we must explicitly preserve the ability for the `ScaleInspectorNotation` component to update the `keySwitches` array when a user clicks the notation arrows.

**Implementation Steps:**

**Part 1: Update `useMidiStore.ts**`

1. **Remove Passive Sync:** Inside `useMidiStore.ts`, remove the line `keySwitches: syncKeySwitches(...)` from the returned state objects in `setRootNote`, `setScaleDecimalId`, and `setActiveState`.
2. **Delete Helper:** Delete the `syncKeySwitches` function block entirely. Keep `getRootString` and `getScaleTypeString`.
3. **Add Explicit Sync Action:** Add this new action to the store (don't forget to update the `MidiStore` interface in `types/midi.ts`):
```typescript
updateActiveKeySwitchFromNotation: (newRoot: number, newDecimal: number) => 
  set((state) => {
    const idx = state.activeState.activeSwitchIndex;
    const updatedSwitches = [...state.activeState.keySwitches];
    updatedSwitches[idx] = {
      root: getRootString(newRoot),
      type: getScaleTypeString(newDecimal)
    };
    return {
      activeState: { 
        ...state.activeState, 
        keySwitches: updatedSwitches 
      }
    };
  }),

```



**Part 2: Update `ScaleInspectorNotation.tsx**`

1. Locate the click handlers for the scale/root adjustment arrows (e.g., `handleRootChange`, `handleScaleChange`).
2. Ensure that alongside updating the active state (via `setRootNote` or `setActiveState`), these handlers also explicitly call the new `updateActiveKeySwitchFromNotation(newRoot, newDecimal)` function to sync the UI array.
