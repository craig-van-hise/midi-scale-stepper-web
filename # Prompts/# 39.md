# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This is a surgical correction for the `ScaleChangeSettingsModal`. Previous implementations resulted in three critical errors:
1. **Misplaced Trigger:** The cog was placed in the parent header rather than the info strip.
2. **Hybrid Styling:** The modal failed to match the "Output Settings" radio-button design.
3. **DOM Trapping:** The modal is currently trapped inside a container with `overflow: hidden` and `scale(0.80)`, causing it to be clipped and rendered incorrectly.
**Objective:** Relocate the trigger, fix the styling to match the reference design, and use a React Portal to render the modal at the document root.

## 2. Technical Decisions & Dependencies
* **React Portals:** The modal must use `createPortal` from `react-dom` to render into `document.body`. This is non-negotiable to fix the clipping/scaling issue.
* **Positioning:** Because the modal is now a Portal, it will default to the center of the viewport (fixed inset-0).
* **Styling Strictness:** Use native radio buttons and remove the "X" button to match the "Output Settings" visual identity.
