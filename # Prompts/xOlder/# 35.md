# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This PRP introduces the **"First Note Exception"** logic to the Scalar Stepper engine. The objective is to enforce that the very first trigger of the Stepper (regardless of the specific interval requested, e.g., +1 or -2) will ignore the step offset and instead play the active "Start Note" (defaulting to C4). 

Additionally, this exception must reset so that the engine announces the root note *first* before stepping whenever:
1. The user presses the "Home" key (A0).
2. The user changes the Scale or Root Note while the global scale change mode is set to "Follow Root".

## 2. Technical Decisions & Dependencies
* **State Flagging:** We will introduce an `isFirstNote` boolean flag to the `activeState` slice in Zustand.
* **Engine Intercept:** Inside `ScaleStepperEngine.ts`, we will read this flag. If `true`, the engine bypasses the index math, sets the output note to the current anchor (`lastPlayedMidi`), and immediately toggles `isFirstNote` to `false`.
* **TypeScript Interfaces:** The agent must update the type definitions in `src/types/midi.ts` to accommodate the new flag and setter.
