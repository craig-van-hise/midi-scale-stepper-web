# Product Requirements Prompt (PRP) — A0 Home Switch Implementation 🔘

## 1. Project Context & Objectives
The application's core MIDI routing and Stepper logic have been validated. However, the functionality for the "Home Switch" zone (MIDI keys 21-23 / A0-B0) was left incomplete. 

Currently, these keys are correctly intercepted (blocked from output), but MIDI note 21 (A0) does not execute its intended function. Pressing A0 must act as a "Home" button, instantly resetting the application's `lastPlayedMidi` (the Start Note) back to the scale's root note, specifically situated in the octave defined by the user's `startingOctave` global setting.

## 2. Technical Decisions & Dependencies
* **Home Note Math:** The exact MIDI note to reset to must be calculated using the `rootNote` (0-11) and the `startingOctave` from the Zustand store.
    * **Formula:** `homeNote = (rootNote % 12) + ((startingOctave + 1) * 12)`
    * *Example:* If root is 2 (D) and starting octave is 3, `2 + (4 * 12) = 50` (D3).
* **Routing Logic:** The intercept for this zone (21-23) must specifically listen for `NoteOn` events on exactly note `21`. Notes `22` and `23` remain functionally dead/swallowed.

## 3. Implementation Phases

### Phase 1: Store Action (`useMidiStore.ts`) — ✅ Completed
* **Objective:** Create the deterministic state mutation for the Home reset.
* **Instructions:**
    1. Open `useMidiStore.ts`.
    2. Add a new action to the store interface: `triggerHomeReset: () => void;`
    3. Implement `triggerHomeReset`. It must read `rootNote` and `startingOctave` from the current state (`get()`).
    4. Calculate the `homeNote` using the exact formula provided in Section 2.
    5. Update the state by setting `lastPlayedMidi` to this newly calculated `homeNote`.
* **TDD Checkpoint:**
    * **Test Case 1:** Given state `{ rootNote: 0, startingOctave: 4 }` (C4). When `triggerHomeReset()` is called, assert `lastPlayedMidi` updates to `60`.
    * **Test Case 2:** Given state `{ rootNote: 7, startingOctave: 2 }` (G2). When `triggerHomeReset()` is called, assert `lastPlayedMidi` updates to `43`.
    * **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Implement the action and write tests in `useMidiStore.test.ts`. Once PASS, set status to ✅ Completed.

### Phase 2: Web MIDI Intercept (`useWebMidi.ts`) — ✅ Completed
* **Objective:** Wire the physical A0 key to the new store action.
* **Instructions:**
    1. Open `useWebMidi.ts`. Locate the intercept block for the Home zone (likely checking for `note < 24` or `note >= 21 && note <= 23`).
    2. Extract `triggerHomeReset` from `useMidiStore`.
    3. Within this zone intercept, check if `note === 21` AND the event is a `NoteOn` (or velocity > 0).
    4. If true, execute `triggerHomeReset()`.
    5. Ensure the function always returns early (drops the note) so notes 21, 22, and 23 never trigger audio or output NoteOn/NoteOff events.
* **TDD Checkpoint:**
    * **Test Case 1:** Simulate incoming MIDI NoteOn for note `21`. Assert `triggerHomeReset` is called, and NO output NoteOn is dispatched.
    * **Test Case 2:** Simulate incoming MIDI NoteOn for note `22`. Assert `triggerHomeReset` is NOT called, and NO output NoteOn is dispatched.
    * **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Wire the intercept and update `useWebMidi.test.ts`. Once PASS, set status to ✅ Completed.

## 4. Final Review & Validation
* **Technical Audit:** Ensure that the A0 key functions strictly as a momentary trigger on press (NoteOn) and that releasing it (NoteOff) does not trigger a second reset.
* **Status Reconciliation:** Verify Phase 1 and 2 are marked ✅ Completed.
