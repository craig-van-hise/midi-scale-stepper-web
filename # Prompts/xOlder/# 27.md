# Product Requirements Prompt (PRP) — UI, Notation & Scale Change Logic 🔘

## 1. Project Context & Objectives
This work order addresses visual discrepancies in the Key Switches, synchronizes the Scale Notation component with the global Stepper engine, fixes broken selection/transpose state within the notation, and introduces a new global "On Scale Change" behavior (Voice-Leading vs. Follow Root). 

The priority is maintaining strict unidirectional data flow. When the notation is clicked, it must drive the global state. When a scale changes, the global state must deterministically update the Start Note before the UI renders.

## 2. Technical Decisions & Dependencies
* **Notation Click-to-Play:** When a note in `ScaleInspectorNotation.tsx` is clicked, it must extract the absolute MIDI pitch, call `setLastPlayedMidi(pitch)`, and fire `triggerOutputNoteOn(pitch, 64)`. A `setTimeout` of `500ms` must follow to fire `triggerOutputNoteOff(pitch)`.
* **On Scale Change Logic:** Add `scaleChangeMode: 'voice-leading' | 'follow-root'` to `useMidiStore`. 
    * **Voice-Leading:** When scale changes, pass the current `lastPlayedMidi` through `RoundingEngine.roundNote` using the *new* scale's bitmask. Set the result as the new `lastPlayedMidi`.
    * **Follow Root:** When scale changes, calculate the home note: `(rootNote % 12) + ((startingOctave + 1) * 12)` and set it as the new `lastPlayedMidi`.
* **Marquee & Transpose Fix:** The selection bug after transposition implies React state mutation. When mapping the transposed notes, the `selectedNotes` array (or equivalent state) must be immutably updated with the new note references, and never explicitly cleared (`[]`) at the end of the `onKeyDown` handler.
