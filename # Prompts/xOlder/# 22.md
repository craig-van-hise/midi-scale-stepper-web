# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The goal of this PRP is to aggressively refactor the Input Keyboard (`KeySplitKeyboard.tsx`), its associated global state (`useMidiStore.ts`), and the physical MIDI routing logic (`useWebMidi.ts`). We are removing dynamic zone ranges (locking them), stripping out legacy Octave knobs/Thru zones, shifting input zones down one octave, and introducing two new functional zones: a "Home" switch (A0) and a mutable "Play/Start Note" zone (C#5-C8) with customizable rounding and audibility settings. Finally, the UI must visually latch the selected keys across all control zones.

## 2. Technical Decisions & Dependencies
* **Locked MIDI Mappings:** The input keyboard zones are now **strictly hardcoded**. You must remove any dynamic resize/drag-handle logic from the Input Keyboard UI.
    * **Zone 1 (Home):** 21 - 23 (A0 - B0) | Color: Brown (Unlabeled)
    * **Zone 2 (Root Select):** 24 - 35 (C1 - B1) | Color: Orange
    * **Zone 3 (Scale Select):** 36 - 47 (C2 - B2) | Color: Yellow
    * **Zone 4 (Stepper):** 48 - 72 (C3 - C5) | Color: Blue
    * **Zone 5 (Play/Start Note):** 73 - 108 (C#5 - C8) | Color: Cyan
* **State Additions:** `useMidiStore.ts` requires a new `playStartSettings` object (`{ audible: boolean; rounded: boolean }`) defaulting to `{ audible: true, rounded: false }`.
* **Calculations (Home Button):** When A0 (21) is pressed, the system must calculate the "Home Note". 
    * Formula: `homeNote = (rootNote % 12) + ((startingOctave + 1) * 12)`. 
    * This result replaces `lastPlayedMidi`.
* **Visual Latching Engine:** The UI needs an array of `activeMidiNotes` to hold keys down.
    * Root Key: `24 + (rootNote % 12)`
    * Scale Key: `36 + activeSwitchIndex`
    * Start Note Key: `lastPlayedMidi`
