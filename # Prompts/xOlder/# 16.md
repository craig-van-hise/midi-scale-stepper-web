## 2. Technical Decisions & Dependencies
* **Data Source Truth:** The stepping offset must NOT be derived from subtracting raw MIDI numbers (e.g., `incomingNote - 72`). It must be derived by looking up the incoming note in `STEPPER_DATA_MAP` and parsing the `index` property.
* **Scalar Math Engine:** Transposition must occur on the *array index* of the scale, not on the semitone value. We will use a deterministic modulo-based algorithm to handle index wrapping and octave shifts simultaneously.

### Phase 2: Scalar Stepping Math & Routing — ✅ Completed
* **Objective:** Implement the mathematical core that adds the `stepOffset` to the current scale index and outputs a physical MIDI note.
* **Instructions:**
    * Create a function `executeScaleStep(stepOffset: number)` that performs the following explicit calculations:
    * **1. Gather State:** Retrieve `lastPlayedMidi`, `rootNote`, and `scaleDecimalId` from `useMidiStore`. Retrieve the `entry` from `getLUT()`.
    * **2. Define Anchor:** `const currentMidi = lastPlayedMidi ?? (60 + rootNote);`
    * **3. Find Current Index:**
        * Calculate the interval from the root: `let currentInterval = (currentMidi - rootNote) % 12;`
        * Ensure positive modulo: `if (currentInterval < 0) currentInterval += 12;`
        * Find the index: `let currentIndex = entry.pitch_classes.indexOf(currentInterval);`
        * Fallback: `if (currentIndex === -1) currentIndex = 0;` (Failsafe if the user changes to a scale that doesn't contain the last played note).
    * **4. Calculate New Index & Octave Shift:**
        * $I_{new} = \text{currentIndex} + \text{stepOffset}$
        * $L = \text{entry.pitch\_classes.length}$
        * `const octaveShift = Math.floor(newIndex / L);`
        * `const wrappedIndex = ((newIndex % L) + L) % L;`
    * **5. Reconstruct MIDI Note:**
        * `const targetInterval = entry.pitch_classes[wrappedIndex];`
        * `const currentOctaveBase = currentMidi - currentInterval;` *(This safely isolates the exact octave anchor of the root)*
        * `const outputMidi = currentOctaveBase + (octaveShift * 12) + targetInterval;`
    * **6. State Push:** Update Zustand `lastPlayedMidi` with `outputMidi`, and route `outputMidi` to your Web MIDI Output function.
    * **UI SYMMETRY:** Update the `triggerStep` stub in `ScaleStepperKeySwitches25.tsx` to call this exact `executeScaleStep(parsedOffset)` function so UI clicks do the exact same thing as MIDI keys.
