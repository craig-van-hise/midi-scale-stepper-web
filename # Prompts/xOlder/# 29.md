# Product Requirements Prompt (PRP) — Follow-Root Logic Fix 🔘

## 1. Project Context & Objectives
The "On Scale Change" logic (Follow-Root) implemented in the previous phase is failing in two ways:
1. **Octave Drift:** The Start Note continuously drifts downward into lower octaves instead of locking to the `startingOctave` global setting.
2. **Erratic Triggering:** The logic does not fire consistently when changing keys.

These are caused by a Javascript negative modulo bug in the pitch calculation, and a failure to bind the evaluation logic to ALL actions that constitute a "Scale Change" (both Scale Type changes and Root Note changes).

## 2. Technical Decisions & Dependencies
* **Positive Wrapping Modulo:** Javascript's `%` operator fails on negative numbers. To find the true Pitch Class (0-11) of the `rootNote`, you MUST use this exact formula: 
  `const pc = ((rootNote % 12) + 12) % 12;`
* **Absolute Home Note Math:** Once you have the safe `pc`, calculate the exact MIDI note using the `startingOctave`:
  `const homeNote = pc + ((startingOctave + 1) * 12);`
* **Universal Trigger:** The evaluation logic (Voice-Leading vs. Follow-Root) must be abstracted into a helper function inside the store, and it MUST be called at the end of **both** the action that sets the Root Note and the action that sets the Scale Index.
