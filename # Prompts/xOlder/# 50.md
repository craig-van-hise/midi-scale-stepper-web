## 1. Project Context & Objectives
The Two-Octave Bidirectional Stepper requires a UI/UX update to better communicate its state to the player. 
1. **Dynamic Labels:** When the keyboard is globally inverted, the UI must visually reflect this math so the user knows what direction the physical keys will play. Positive labels (`+2`, `+Oct`) must become negative (`-2`, `-Oct`), and vice versa.
2. **Invert Key Disambiguation:** The two invert utility keys currently use the identical `<ArrowUpDown>` icon, making it impossible to tell the Momentary switch apart from the Toggle switch at a glance. We need to add visual text cues (e.g., "Tog" and "Hold") beneath their respective icons.

## 2. Technical Decisions & Dependencies
* **Unidirectional State Consumption:** The store already correctly handles the XOR inversion math for the engine: `isInverted = toggle !== momentary`. The UI component simply needs to read this derived boolean state and use it to format the string labels at render-time. 
* **Safe String Manipulation:** To flip labels dynamically, we will use simple string replacement on `action.label` (swapping `+` for `-` and `-` for `+`) rather than calculating mathematical offsets in the UI, keeping the component as a "dumb renderer".
* **DOM Structure Integrity:** We will nest the icons and disambiguation text inside a flex column `div` to prevent disrupting the strict `getWhiteStyle` / `getBlackStyle` absolute positioning bounds.
