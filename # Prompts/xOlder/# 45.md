# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The user has identified two UI bugs regarding the Root and Scale Type dropdown menus within the `ScaleKeySwitches12` component:
1. **Translucency:** The dropdowns inherit a 75% opacity because they are nested inside a span with the `hover:opacity-75` utility class.
2. **Clipping:** The dropdowns are clipped by the parent `KeySwitchContainer`'s `transform: scale()` and `overflow: hidden` context. 

**Objective:** Extract the dropdown menus from the trigger elements to fix the opacity inheritance, and wrap them in a React Portal injected into `document.body` so they transcend all parent clipping contexts.

## 2. Technical Decisions & Dependencies
* **React Portals:** We will use `createPortal` from `react-dom` to render the menus at the root of the DOM.
* **Coordinate Calculation:** Because the menus are now Portals, `position: absolute` with `top: 100%` will no longer anchor them to the info strip. We must capture the exact screen coordinates using `getBoundingClientRect()` when the user clicks the trigger, and apply those absolute coordinates to the portal.

## 3. Implementation Phases

### Phase 1: Coordinate State & Excision — ✅ Completed
* **Objective:** Setup local state for screen coordinates and remove the dropdowns from the nested span.
* **Instructions:**
    1.  Open `src/components/ScaleKeySwitches12.tsx`.
    2.  Import `createPortal` from `react-dom`.
    3.  Add two new state variables for coordinates:
        ```typescript
        const [rootMenuCoords, setRootMenuCoords] = useState({ top: 0, left: 0 });
        const [typeMenuCoords, setTypeMenuCoords] = useState({ top: 0, left: 0 });
        ```
    4.  Update the Root `<span>` onClick handler to capture coordinates:
        ```typescript
        onClick={(e) => {
          const rect = e.currentTarget.getBoundingClientRect();
          // Anchor to bottom center of the trigger
          setRootMenuCoords({ 
            top: rect.bottom + window.scrollY + 8, 
            left: rect.left + window.scrollX + (rect.width / 2) 
          });
          setIsRootMenuOpen(!isRootMenuOpen);
          setIsTypeMenuOpen(false);
        }}
        ```
    5.  Update the Type `<span>` onClick handler to capture coordinates:
        ```typescript
        onClick={(e) => {
          const rect = e.currentTarget.getBoundingClientRect();
          // Anchor to bottom right of the trigger
          setTypeMenuCoords({ 
            top: rect.bottom + window.scrollY + 8, 
            left: rect.right + window.scrollX 
          });
          setIsTypeMenuOpen(!isTypeMenuOpen);
          setIsRootMenuOpen(false);
        }}
        ```
