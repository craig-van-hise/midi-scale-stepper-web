# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
The `StepperContextMenu` portal has severe UX and event-bubbling bugs. 
1. **Event Bleed-Through:** Keyboard events inside the portal are bubbling up the React tree, triggering global application hotkeys (like deleting notation notes or preventing the typing of minus signs).
2. **Redundant Labels:** The `label` input field is bad UX. Labels must be strictly auto-derived from the action type and value.
3. **Broken Submission:** Pressing 'Enter' inside the input does not save the form.

## 2. Technical Decisions & Dependencies
* **Event Trapping:** The outermost `div` of the portal MUST include `onKeyDown={(e) => e.stopPropagation()}`. This stops React from bubbling keystrokes back to the global app listeners.
* **Auto-Derived Labels:** The `label` property will be generated by a strict switch statement right before saving to the store. The user will never type a label.
* **Form Submission:** Pressing `Enter` while focused inside the menu must trigger the save action.

## 3. Implementation Phases

### Phase 1: Label Auto-Generation Logic — ✅ Completed
* **Objective:** Remove the label input from local state and auto-generate it upon saving.
* **Instructions:**
  1. In `src/components/StepperContextMenu.tsx`, remove the `label` string from the component's local state and remove its `<input>` field from the render entirely.
  2. Create a helper function inside the file:
     ```typescript
     const generateLabel = (type: StepperActionType, value: number): string => {
       switch (type) {
         case 'STEP': return value > 0 ? `+${value}` : `${value}`;
         case 'OCTAVE': return value > 0 ? '+Oct' : '-Oct';
         case 'HOME': return 'Home';
         case 'INVERT_TOGGLE': return 'Inv(T)';
         case 'INVERT_MOMENTARY': return 'Inv(M)';
         case 'REPEAT_LAST': return 'Repeat';
         case 'CUSTOM': return 'Open';
         default: return '';
       }
     };
     ```
  3. Update the "Save" handler to calculate the label immediately before calling the store:
     `const finalLabel = generateLabel(type, value);`
     `useMidiStore.getState().updateStepperConfig(index, { type, value, label: finalLabel });`
