# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
We are introducing a dynamic Input Keyboard configuration to support both 88-key and 49-key physical MIDI controllers. 
The 49-key layout intentionally truncates the lower octave of the 24-key Stepper, utilizing only the top octave (indices 12-23) to preserve positive steps and right-hand utility functions. We will add an `InputSettingsModal` to toggle this state, shift the engine's MIDI routing boundaries dynamically, and strictly center the visual 49-key layout to maintain physical key dimensions without CSS stretching.

## 2. Technical Decisions & Dependencies
* **Dynamic Zone Math:** The engine must route MIDI messages differently based on the active `inputKeyboardSize`.
  * **88-Key Mode (Legacy):**
    * Root Select: `24-35` (C1-B1)
    * Scale Select: `36-47` (C2-B2)
    * Stepper: `48-71` (C3-B4) 
    * Play/Start: `72+` (C5+)
  * **49-Key Mode:**
    * Root Select: `36-47` (C2-B2)
    * Scale Select: `48-59` (C3-B3)
    * Stepper: `60-71` (C4-B4) — *Crucially, it still uses `midiNote - 48` to correctly target indices 12-23 in the config.*
    * Play/Start: `72+` (C5+)
* **Visual Centering:** `KeySplitKeyboard.tsx` must use flexbox centering so the 49-key array floats in the middle of the card rather than anchoring to the left or stretching.
