# Product Requirements Prompt (PRP) — Scale Notation State Restoration 🔘

## 1. Project Context & Objectives
In the previous update to `ScaleInspectorNotation.tsx`, the stable React state mutations from the original `rect-midi-components` project were improperly replaced with imperative reads to force Zustand syncing. This created stale closures in the keyboard event listeners, causing the transpose feature (Arrow Up/Down) to break after two uses, and causing the multi-note selection to clear unexpectedly.

We must immediately restore the functional state updater pattern (`setPitches(prev => ...)`) from the original file, use `useRef` to maintain stable references for the event listeners, and safely decouple the Zustand synchronization.

## 2. Technical Decisions & Dependencies
* **Stable References (Refs):** To prevent the `keydown` event listener from constantly tearing down and rebuilding, we will introduce `pitchesRef` and `selectedIndicesRef`. The keyboard listener will read from these refs to guarantee it always has the latest state without causing React re-renders.
* **Functional Mutation:** `mutateSelected` must return to the original pattern of using `setPitches(prev => ...)`.
* **Zustand Sync Decoupling:** We will use a single `useEffect` to watch the local `pitches` array. Whenever it changes, we calculate the bitmask. If the bitmask differs from `useMidiStore.getState().activeState.scaleDecimalId`, we push the update to the store. This prevents the "selection clearing" race condition.
