# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
**BUG FIX:** The 49-key layout correctly centers the physical keys, but the colored Zone Header cards ("ROOT SELECT", "SCALE SELECT", "STEPPER", "PLAY/START NOTE") remain misaligned. They are still using the hardcoded widths and left-justification of the 88-key layout. 

**Objective:** Refactor the Zone Header row in `KeySplitKeyboard.tsx` to be dynamically driven by the `inputKeyboardSize` state. The header container must be perfectly aligned with the centered keys, and the individual banner widths must mathematically match the number of white keys in each zone.

## 2. Technical Decisions & Dependencies
* **Dynamic Width Math:** The physical width of a zone banner is calculated by multiplying the number of **white keys** in that zone by the standard white key CSS width (usually ~`40.5px` or `40px` in your layout).
  * **88-Key Mode Zone Spans:** Root (7 white keys), Scale (7 white keys), Stepper (14 white keys), Play/Start (24 white keys).
  * **49-Key Mode Zone Spans:** Root (7 white keys), Scale (7 white keys), Stepper (7 white keys), Play/Start (8 white keys).
* **Flex Alignment:** The container holding the header banners MUST be wrapped in the same `justify-content: center` flex structure as the keys below it to ensure they move together as a single cohesive unit.

### Phase 1: Dynamic Header Widths — ✅ Completed

* **Instructions:**

  3. Define the white-key counts for each zone dynamically based on the size:

     const scaleWhiteKeys = 7; // Always 7 (C3-B3 for 49, C2-B2 for 88... wait, both are 1 octave, so always 7)
