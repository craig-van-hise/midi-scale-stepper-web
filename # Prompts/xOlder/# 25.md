# Product Requirements Prompt (PRP) — Playback & Note Hang Fixes 🔘

## 1. Project Context & Objectives
The recent implementation introduced three critical playback bugs that must be resolved:
1. **Audio Context Suspension:** The newly added synth tone does not play via physical MIDI input until a UI element is clicked, due to the browser's autoplay policy.
2. **UI Note Hangs:** Clicking the Stepper UI keys triggers a note but leaves it hanging indefinitely (missing NoteOff dispatch).
3. **Polyphony State Collision:** Holding a Stepper input, playing a Play/Start input, and then releasing the Stepper input causes the Stepper output note to hang. The NoteOff calculation is using mutated state instead of the original offset.

## 2. Technical Decisions & Dependencies
* **Active Note Registry:** To solve the polyphony state collision, we must NEVER recalculate math on a `NoteOff` event. 
    * We will introduce a local map: `const activeNotes = useRef(new Map<number, number>())` inside `useWebMidi.ts`.
    * **Key =** `rawMidiInputNote`
    * **Value =** `calculatedOutputNote`
    * On `NoteOn`: Calculate output -> save to map -> dispatch `NoteOn`.
    * On `NoteOff`: Retrieve output from map -> dispatch `NoteOff` -> delete from map.
* **AudioContext Resume:** The synth engine must bind a one-time global event listener (e.g., `pointerdown` or `keydown` on the `window`) to force `AudioContext.resume()` so physical MIDI can play sound immediately after the user interacts with the page.

## 3. Implementation Phases

### Phase 1: Polyphony / Active Note Registry (`useWebMidi.ts`) — ✅ Completed
* **Objective:** Ensure `NoteOff` events always terminate the exact MIDI note that was generated by their corresponding `NoteOn`, regardless of intermediate state changes.
* **Instructions:**
    1. Open `useWebMidi.ts`. Inside the hook, instantiate a tracking map: `const activeNotesRegistry = useRef(new Map<number, number>());`
    2. In the **Play/Start Zone** intercept (73-108):
        * *NoteOn:* Calculate `targetNote`. `activeNotesRegistry.current.set(note, targetNote)`. Dispatch NoteOn.
        * *NoteOff:* Look up `targetNote = activeNotesRegistry.current.get(note)`. If found, dispatch NoteOff for `targetNote`, then `activeNotesRegistry.current.delete(note)`.
    3. In the **Stepper Zone** intercept (48-72):
        * Apply the exact same logic. Calculate the final output note on `NoteOn`, store it in the registry using the raw input `note` as the key, and retrieve it on `NoteOff` to terminate it.
