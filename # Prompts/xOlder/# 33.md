# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This PRP addresses a series of global UI layout corrections and default state adjustments for the MIDI Scale Stepper MVP. The objective is to center the main application layout to eliminate right-side whitespace on wide screens, wire the visual global bypass ("Power") state to the main interface, and update the default initialization parameters for the Output Filter engine (setting bounds to C2-B5 and mode to Smart Wrap).

## 2. Technical Decisions & Dependencies
* **MIDI Math Constraints:** The requested default range is C2 to B5. In standard MIDI notation (where Middle C is C4 = 60), **C2 is 36** and **B5 is 83**.
* **CSS Layout Architecture:** The application currently uses `items-start` on the `<main>` flex container, forcing all full-width child cards to the left. Changing this to `items-center` will properly align the stack within the `max-w-[1600px]` boundary.
* **Global Power State Handling:** The `Header.tsx` already toggles `globalSettings.power`. To visually disable the app without blocking the header, the `<main>` container in `App.tsx` must read this state and dynamically apply Tailwind utility classes (`opacity-50 pointer-events-none select-none transition-all`) to disable all physical interactions and visually shade the app.
