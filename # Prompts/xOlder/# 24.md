
# Product Requirements Prompt (PRP) — Stepper Math Course Correction 🔘

## 1. Project Context & Objectives
The Play/Start Note zone functionality (audible passthrough and momentary switching) has been **VALIDATED**. 

However, the Stepper Zone (keys 48 - 72) is currently calculating offsets chromatically (-12 to +12). This is incorrect. The Stepper must use a **fully dynamic algorithm** that calculates the exact number of valid *scale steps* between the C4 anchor (MIDI `60`) and the played note, dynamically adjusting based on the `activeScaleBitmask` and the global `roundingPreference`. 

We must rebuild the Stepper math and UI to reflect these dynamic diatonic steps.

## 2. Technical Decisions & Dependencies
* **Dynamic Step Algorithm (`calculateDynamicStepOffset`):** You must implement this logic.
  1. The anchor is always `60` (C4), representing an offset of `0`.
  2. Round the `rawMidi` input using `RoundingEngine.roundNote` based on the active scale and rounding preference.
  3. Count the number of valid scale notes (using `BitmaskCalculator.isNoteInBitmask`) between `60` and the rounded target. 

```typescript
// Implement this in utils/ScaleStepperEngine.ts (or equivalent math utility)
export function calculateDynamicStepOffset(
  rawMidi: number, 
  scaleBitmask: number, 
  roundingMode: 'up' | 'down'
): number {
  if (rawMidi === 60) return 0;

  const roundedTarget = RoundingEngine.roundNote(rawMidi, scaleBitmask, roundingMode);
  if (roundedTarget === 60) return 0;

  let steps = 0;
  if (roundedTarget > 60) {
    for (let i = 61; i <= roundedTarget; i++) {
      if (BitmaskCalculator.isNoteInBitmask(i % 12, scaleBitmask)) steps++;
    }
    return steps;
  } else {
    for (let i = 59; i >= roundedTarget; i--) {
      if (BitmaskCalculator.isNoteInBitmask(i % 12, scaleBitmask)) steps++;
    }
    return -steps;
  }
}

```
