
# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
We are revamping the current MIDI Scale Stepper UI and internal engine routing to support a **Two-Octave Bidirectional Stepper** layout. The existing `-7 to +7` layout is being deprecated in favor of a 24-key performance layout strictly spanning **C3 to B4**. 

This update introduces complex performance utility keys (Invert, Home, Repeat Last, Octave jumps) mirrored symmetrically on the outer edges of the keyboard, anchored by a `0` step at C4. It also necessitates shifting the physical MIDI zone mappings to accommodate the expanded 24-key Stepper Zone without colliding with the Scale Select Zone.

## 2. Technical Decisions & Dependencies
To enforce our strict unidirectional architecture and zero-ambiguity execution, you must adhere to the following logic:

* **Explicit Override (24 Keys Only):** The provided design brief mentions a 25th key (C5). **You must ignore this.** The user has explicitly overridden this requirement. The layout is strictly **24 keys**, index 0 to 23, mapping to C3 through B4.
* **Inversion Math (XOR Logic):** The engine requires global vector inversion. 
  * `isInverted = uiState.stepperInvertToggle !== uiState.stepperInvertMomentary`
  * When `isInverted === true`, all `STEP` and `OCTAVE` values must be multiplied by `-1` before being passed to the `ScaleStepperEngine`.
* **Octave Action Math:** A "Scale Octave" is dynamic based on the active scale. The store must calculate the step amount dynamically based on the current scale's length. `octaveStep = scale.length * direction`.
* **Action Data Structure:** The Stepper keys are no longer just integers. They must be data-driven objects to support the UI right-click menu in the future. You will implement this type:
  ```typescript
  export type StepperActionType = 'STEP' | 'OCTAVE' | 'INVERT_TOGGLE' | 'INVERT_MOMENTARY' | 'HOME' | 'REPEAT_LAST' | 'CUSTOM';
  export interface StepperAction {
      type: StepperActionType;
      value: number; // e.g., 1 for STEP, 1 for OCTAVE (direction), or 0 for toggles
      label: string;
  }

```

* **MIDI Zone Realignment:** To fit C3-B4 (MIDI 48-71) for the Stepper, you must shift the existing Scale Select zone down to C2-B2 (MIDI 36-47).

### Phase 2: Engine Routing & Action Logic — 🔘 Pending

* **Objective:** Implement the logic router in the store that processes these actions safely.
* **Instructions:**
1. In `useMidiStore.ts`, create a new action: `triggerStepperAction(keyIndex: number, isNoteOn: boolean)`.
2. **Logic Tree for NoteOn:**
* Extract the `StepperAction` from `stepperKeyMap[keyIndex]`.
* **If `INVERT_TOGGLE`:** Flip `stepperInvertToggle` state.
* **If `INVERT_MOMENTARY`:** Set `stepperInvertMomentary` to true.
* **If `HOME`:** Reset the active scale pitch pointer to the root note (Step 0) and trigger output.
* **If `STEP` or `OCTAVE` or `REPEAT_LAST`:**
* Calculate `effectiveAction` (resolve `REPEAT_LAST` to `lastStepperAction`, abort if null).
* Determine base step value. If `OCTAVE`, multiply action value by `currentScale.length`.
* Apply XOR Inversion: If `toggle !== momentary`, multiply the final step value by `-1`.
* Save to `lastStepperAction` (only if it wasn't a `REPEAT_LAST` call).
* Pass the final step integer to the existing `ScaleStepperEngine`.




3. **Logic Tree for NoteOff:**
* Extract the `StepperAction`.
* **If `INVERT_MOMENTARY`:** Set `stepperInvertMomentary` to false.
* Do nothing for others.




* **TDD Checkpoint:**
* **Test Case 1 (XOR Math):** Given toggle=true and momentary=false. When triggering STEP index 14 (+2), Assert the engine receives `-2`.
* **Test Case 2 (Double Invert):** Given toggle=true. When Momentary NoteOn is triggered, Assert inversion is bypassed (engine receives positive). When Momentary NoteOff, Assert inversion returns.
* **Test Case 3 (Octave Math):** Given a 5-note pentatonic scale. When triggering OCTAVE index 19 (+1), Assert the engine receives `+5`.
