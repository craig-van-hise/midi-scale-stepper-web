# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**UI REFINEMENT:** The Input card is currently too tall vertically compared to the slimmer Output card. While the Octave Knob is now correctly centered above the Play/Start banner, the combined vertical padding of the card container and the margins separating the header from the keyboard assembly have created excessive dead space.

**Objective:** Slim down the vertical footprint of the Input card by tightening the container padding (especially bottom padding) and reducing the margin above the keyboard assembly to the absolute minimum required to clear the floating Octave Knob.

## 2. Technical Decisions & Dependencies
* **Padding vs. Margin:** The height of the card is dictated by three factors: the top padding of the card, the bottom padding below the keys, and the `marginTop` or `gap` separating the header row from the keyboard wrapper. We must systematically reduce all three to compress the card's height.
* **Component Targets:** This requires adjusting styles in the parent component that renders the Input card (likely `App.tsx` or a layout container) as well as the wrapper inside `KeySplitKeyboard.tsx`.

## 3. Implementation Phases

### Phase 1: Global Card Padding Reduction — ✅ Completed
* **Objective:** Compress the internal padding of the Input card container.
* **Instructions:**
  1. Locate the component rendering the physical "Input" card wrapper (the white box with the drop shadow). This is likely in your main `App.tsx` or a layout file.
  2. Inspect its padding classes (e.g., `p-6` or `py-6`). 
  3. Reduce the vertical padding. Change `py-6` to `py-3` or `pt-3 pb-2`. Ensure the bottom padding (`pb`) is exceptionally tight, as the physical keys do not need much breathing room at the bottom of the card.
* **TDD Checkpoint:**
  * **Test Case 1:** Inspect the DOM tree to assert the Input card's parent container has reduced vertical padding classes.
  * **AGENT INSTRUCTION:** "Set Phase status to 🔵 Active. Apply the padding reductions. Execute tests. Once they PASS, set status to ✅ Completed."

### Phase 2: Internal Margin Tightening — ✅ Completed
* **Objective:** Reduce the space between the card header ("Input" text) and the keyboard banners.
* **Instructions:**
  1. In `src/components/KeySplitKeyboard.tsx`, locate the main wrapper `div` that holds the banners and the keys.
  2. In the previous PRP, we added a specific `marginTop` (e.g., `mt-10` or `marginTop: '40px'`) to clear the knob. 
  3. Dial this margin back significantly (e.g., change `mt-10` to `mt-6`, or `40px` to `24px`). The Octave Knob is allowed to sit very close to the horizontal plane of the "Input" title text to save space.
  4. Ensure the knob's `marginBottom` (the space between the knob and the cyan banner) remains tight (e.g., `4px` or `6px`).
* **TDD Checkpoint:**
  * **Test Case 1:** Inspect the `KeySplitKeyboard` wrapper's margin class/style to ensure it has been reduced.
  * **AGENT INSTRUCTION:** "Set Phase status to 🔵 Active. Update the internal margins. Execute tests. Once they PASS, set status to ✅ Completed."
