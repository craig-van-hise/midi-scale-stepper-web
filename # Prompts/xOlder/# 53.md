# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
We have two final cleanup tasks for the 24-key Two-Octave Stepper migration:
1. **Right-Click / Ctrl-Click Event Blocking:** The physical stepper UI keys are currently triggering a "play" (NoteOn) event when right-clicked to open the context menu. We must prevent `onPointerDown` from executing its audio/engine logic if the interaction is a context menu trigger.
2. **Input Keyboard Zone Shift:** Because the Stepper Zone was reduced to end at B4 (MIDI 71), the adjacent "Play/Start Note" zone needs to shift one half-step left. It will now begin at C5 (MIDI 72) instead of C#5 (MIDI 73), sealing the gap on the visual Input Keyboard.

## 2. Technical Decisions & Dependencies
* **Pointer Event Gating:** React's `onPointerDown` provides the `e.button` property (where `0` is left-click, `2` is right-click) and the `e.ctrlKey` boolean (critical for Mac users who use Ctrl+Click for context menus). The NoteOn handler must return early if either of these conditions are met.
* **Zone Boundary Integrity:** The upper boundary of the Stepper zone is `71`. The Play/Start zone must catch `72` and above. This logic likely lives in `src/hooks/useWebMidi.ts` and the visual definitions in `src/components/KeySplitKeyboard.tsx`.
