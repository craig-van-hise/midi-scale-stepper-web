# Product Requirements Prompt (PRP) — Course Correction 1 ✅ Completed

## 1. Project Context & Objectives
The previous implementation of the new Input Keyboard zones failed at the signal routing layer. The Stepper zone logic is still mapped to the legacy octave, causing zone collisions with the new Play/Start Note zone. Furthermore, the Play/Start Note zone is failing to pass MIDI notes to the output when `audible` is true, and it is entirely ignoring out-of-scale notes instead of applying the global `RoundingEngine`. 

This course correction will rigidly enforce the mathematical offsets, fix the note-on/note-off passthrough, and properly integrate the rounding logic.

## 2. Technical Decisions & Dependencies
* **Stepper Offset Math:** The Stepper zone is now strictly `48 - 72` (C3 - C5). The "zero offset" center is C4 (MIDI 60). The calculation for step offsets must be updated everywhere to: `stepOffset = incomingNote - 60`.
* **Play/Start Routing (73 - 108):** * When a note enters this zone, it must ALWAYS evaluate what the rounded note *would* be using the existing `RoundingEngine` and the global rounding preference (Round Up vs Round Down).
    * If `playStartSettings.rounded === true`, the system uses the `roundedNote`. If `false`, it uses the `rawNote`.
    * The selected note updates `lastPlayedMidi`.
    * If `playStartSettings.audible === true`, the system MUST physically dispatch the selected note to the output (both `NoteOn` and `NoteOff` events).
