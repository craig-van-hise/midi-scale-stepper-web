# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This PRP addresses two specific bugs regarding the interaction between the Scale Notation component, the Scale Select keyboard, and the global Zustand store:
1. **Notation Arrow Down Bug:** Pressing the Down Arrow transposes the selected note correctly but unintentionally shifts the active selection to the adjacent note.
2. **Key Switch Preset Desync:** Changing the active scale root or type (either via the physical Root Select keyboard or via the Notation UI) successfully updates the global `activeState.scaleDecimalId` and `rootNote`, but fails to overwrite the data inside the `keySwitches` array at the currently active index. This causes the Key Switch UI to act "stale," rather than functioning as live user presets.

## 2. Technical Decisions & Dependencies
* **Arrow Down Bug Fix:** In `ScaleInspectorNotation.tsx`, the `ArrowDown` event does correctly call `e.preventDefault()`, but the internal logic of `mutateSelected(-1)` is bleeding into subsequent evaluations (likely due to React event pooling or external focus traps). However, since ArrowUp works flawlessly, the issue lies in a subtle index shift occurring elsewhere. We will explicitly enforce selection retention inside the `mutateSelected` callback by refreshing the selection immediately after mutation.
* **Store Write-Back Logic:** To fix the preset syncing, we must update the `setRootNote`, `setScaleDecimalId`, and `setActiveState` actions in `useMidiStore.ts`. Whenever the global root or scale ID changes, the store must calculate the new spelling (Root string and Type string), and explicitly mutate the `keySwitches` array at `state.activeState.activeSwitchIndex`.
* **String Spelling Dependencies:** The store will need access to standard notation mapping (e.g., mapping `rootNote` integer to "C", "C#") and mapping the decimal ID back to a known scale type string. We will import `NOTE_TO_PC` (reversed) or write a small internal map, and we will import `getLUT()` to retrieve the `scale_type` string.

### Phase 2: Implement Key Switch Write-Back in Store — ✅ Completed
* **Objective:** Update `useMidiStore.ts` to sync `activeState` changes back into the `keySwitches` preset array.
* **Instructions:**
    1.  Open `src/store/useMidiStore.ts`.
    2.  Add a helper function at the top of the file to resolve string names:
        ```typescript
        const getRootString = (rootNote: number): string => {
          const roots = ['C', 'C#', 'D', 'D#', 'E', 'F', 'F#', 'G', 'G#', 'A', 'A#', 'B'];
          return roots[((rootNote % 12) + 12) % 12];
        };
        const getScaleTypeString = (decimalId: number): string => {
          const lut = getLUT();
          return (lut && lut[decimalId]) ? lut[decimalId].scale_type : 'Major';
        };
        ```
    3.  Create a helper function to generate the updated `keySwitches` array:
        ```typescript
        const syncKeySwitches = (state: any, newRoot?: number, newDecimal?: number) => {
          const idx = state.activeState.activeSwitchIndex;
          const currentRoot = newRoot !== undefined ? newRoot : state.activeState.rootNote;
          const currentDecimal = newDecimal !== undefined ? newDecimal : state.activeState.scaleDecimalId;
          
          const updatedSwitches = [...state.activeState.keySwitches];
          updatedSwitches[idx] = {
            root: getRootString(currentRoot),
            type: getScaleTypeString(currentDecimal)
          };
          return updatedSwitches;
        };
        ```
    4.  Update the `setRootNote` action: 
        Inject `keySwitches: syncKeySwitches(state, note, undefined)` into the returned `activeState` object.
    5.  Update the `setScaleDecimalId` action:
        Inject `keySwitches: syncKeySwitches(state, undefined, decimalId)` into the returned `activeState` object.
    6.  Update the `setActiveState` action:
        Inject `keySwitches: syncKeySwitches(state, activeState.rootNote, activeState.scaleDecimalId)` into the returned `nextActive` object before returning.
