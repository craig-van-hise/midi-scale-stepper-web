## 1. Project Context & Objectives
The Two-Octave Bidirectional Stepper (24-key layout) is functionally stable. We must now implement the **Right-Click Customization Override**. Users must be able to right-click any of the 24 physical keys to open a context menu, allowing them to change the key's assigned `StepperActionType`, numeric value, and display label. 

## 2. Technical Decisions & Dependencies
* **React Portals (Strict Architecture Rule):** The context menu MUST be built using `ReactDOM.createPortal` and injected into `document.body`. It will receive absolute `X` and `Y` screen coordinates from the `onContextMenu` synthetic event. This ensures the dropdown escapes any parent `overflow: hidden` or `transform: scale()` CSS properties.
* **Controlled Forms:** The menu will act as a controlled React component reading from local state until the user clicks "Save", at which point it fires a single mutation to the Zustand store.
* **Click-Outside Logic:** The menu must automatically close if the user clicks anywhere outside of its DOM node.
