# Product Requirements Prompt (PRP) — Notation Audio Routing Fix 🔘

## 1. Project Context & Objectives
The newly implemented "click to play" feature in the `ScaleInspectorNotation.tsx` component is incorrectly routing its visual feedback to the Input Keyboard. When a user clicks a note on the staff, it should only light up the Output Keyboard and send hardware MIDI out, but it is currently adding the note to the `activeKeys` array (which governs the Input UI).

We must isolate this routing so it exclusively uses `outputActiveKeys`.

## 2. Technical Decisions & Dependencies
* **Strict State Separation:** The `addActiveKey` and `removeActiveKey` actions are strictly reserved for physical MIDI input coming from `useWebMidi`. Virtual clicks from the Notation component must NEVER touch these actions. They must only use `addOutputKey` and `removeOutputKey`.

## 3. Implementation Phases

### Phase 1: Output Routing Fix (`ScaleInspectorNotation.tsx`) — 🔘 Pending
* **Objective:** Remove the Input Keyboard state mutations from the local output routing helper.
* **Instructions:**
    1. Open `src/components/ScaleInspectorNotation.tsx`.
    2. Locate the `outputRouting` object definition at the top of the file.
    3. Remove the calls to `addActiveKey` and `removeActiveKey`. The object must look exactly like this:
    ```typescript
    export const outputRouting = {
      triggerOutputNoteOn: (pitch: number, velocity: number = 64) => {
        useMidiStore.getState().addOutputKey(pitch);
        // Note: If physical MIDI out is handled globally via a listener on outputActiveKeys, 
        // this is sufficient. Otherwise, ensure hardware dispatch happens here.
      },
      triggerOutputNoteOff: (pitch: number) => {
        useMidiStore.getState().removeOutputKey(pitch);
      }
    };
    ```
