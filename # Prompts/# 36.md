## 1. Project Context & Objectives
A recent update introduced an `isFirstNote` flag to govern the Stepper Engine's "First Note Exception" logic. However, this flag was incorrectly coupled to the A0 (Home) key's reset action. The objective of this corrective task is to surgically remove the `isFirstNote: true` reset from the `triggerHomeReset` action in the Zustand store, ensuring the A0 key only snaps the anchor note and does *not* trigger the first-note bypass logic.

## 2. Technical Decisions & Dependencies
* **Targeted Rollback:** We are only altering `src/store/useMidiStore.ts`. The UI components, Stepper Engine (`executeScaleStep`), and the "Follow Root" reset logic must remain completely untouched.
* **State Preservation:** `triggerHomeReset` must be reverted to its previous behavior, updating only `lastPlayedMidi` while leaving `isFirstNote` exactly as it was prior to the action call.
