## 1. Project Context
**Project Name:** MIDI Scale Stepper MVP
**Description:** A web-based MIDI manipulation and routing application. Its primary function is to intercept raw MIDI input and route it through a highly deterministic "Scalar Stepper" engine. Instead of transposing incoming notes by raw semitones, it steps through the specific indices of musical scales using complex interval math, octave wrapping, and boundary filtering before sending the processed MIDI to an output device.

## 2. Core Architecture & Signal Flow
The application follows a strict unidirectional data flow:
* **State Management:** Handled via Zustand (`useMidiStore`). We strictly avoid passive `useEffect` syncs; all state mutations are driven by explicit user events (MIDI Note On, UI Clicks).
* **The PCS_LUT:** Scales are stored in a binary Look-Up Table (`PCS_LUT.dat`) and parsed into memory. Scales are zero-normalized to C, and resolved using a 12-bit decimal integer (bitmask).
* **Signal Pipeline:**
  1. **Input:** `useWebMidi` intercepts raw MIDI.
  2. **Scale Select Zone (C3-B3 / 48-59):** Updates the global `activeSwitchIndex`, changing the active scale in the engine.
  3. **Stepper Zone (C4-C6 / 60-84):** Triggers the Stepper Engine. Looks up a step offset (e.g., `+1`, `-2`) from `STEPPER_DATA_MAP`.
  4. **Processing:** The engine maps the `lastPlayedMidi` to the active scale index, adds the offset, applies modulo wrapping, and resolves the new pitch class.
  5. **Output Filtering:** The final note is passed through `applyOutputFilter` (`octave_wrap` or `smart_wrap`) to strictly constrain it within the user's defined output range before firing.
