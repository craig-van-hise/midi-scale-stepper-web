# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The user has identified a routing bug where physical MIDI inputs in the "Play/Start Note" zone (MIDI 73 - 108) are bypassing the global output filter. Currently, only notes generated by the Stepper engine are being filtered. The objective is to import and apply the `applyOutputFilter` function inside the Play/Start intercept block in `useWebMidi.ts` so that all starting notes obey the user's `smart_wrap` or `octave_wrap` constraints before being routed to the output device.

## 2. Technical Decisions & Dependencies
* **Engine Reusability:** We will utilize the existing `applyOutputFilter` function exported from `src/utils/ScaleStepperEngine.ts`.
* **State Intercept:** If `applyOutputFilter` returns `null` (meaning the note was dropped due to strict octave wrapping bounds), the engine must gracefully abort routing that specific note to `activeNotesRegistry`, `lastPlayedMidi`, and `outputActiveKeys`, preventing silent hanging notes or out-of-bounds UI renders.
* **Strict State Access:** As established in previous PRPs, all state access must use `useMidiStore.getState()` rather than relying on stale closure variables.

## 3. Implementation Phases

### Phase 1: Apply Filter to Play/Start Zone — ✅ Completed
* **Objective:** Route the calculated Play/Start note through the filter engine in `useWebMidi.ts`.
* **Instructions:**
    1.  Open `src/hooks/useWebMidi.ts`.
    2.  Ensure `applyOutputFilter` is imported from `../utils/ScaleStepperEngine`.
    3.  Locate the Play/Start Note Zone Note On block: `if (note >= 73 && note <= 108) { ... }`.
    4.  Extract the filter settings from the state: `const { filterMode, filterRange } = state.globalSettings;`.
    5.  Directly after calculating `const targetNote = rounded ? roundedNote : rawNote;`, apply the filter:
        ```typescript
        const finalTargetNote = applyOutputFilter(targetNote, filterMode, filterRange[0], filterRange[1]);
        
        if (finalTargetNote !== null) {
          state.setLastPlayedMidi(finalTargetNote);
          activeNotesRegistry.current.set(note, finalTargetNote);
          
          if (audible) {
            state.addActiveKey(finalTargetNote);
            state.addOutputKey(finalTargetNote);
          }
        }
        
        // Always highlight the actual physical key pressed on the input UI
        state.addActiveKey(note);
        return;
        ```
    6.  Ensure that the existing logic pushing `targetNote` directly to the registry and state is completely replaced by this new `finalTargetNote` block.
